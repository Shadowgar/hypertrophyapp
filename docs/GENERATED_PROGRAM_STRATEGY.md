# Generated Program Strategy

Last updated: 2026-04-27
Status: active strategy for `Make me a plan`

## Grand Vision
`Make me a plan` should become a deterministic coaching product that produces practical, scalable hypertrophy plans from compiled doctrine + policy + user state, with no runtime LLM dependency.

## Path Separation
- Authored path (`phase_1`, `phase_2`): productized fixed programs, stable and maintained for regressions.
- Generated path (`full_body_v1` and successors): deterministic prescription path that adapts planning logic from user profile and logs.

Authoring and generation are both first-class, but generated planning is the active build focus.

## Status Summary (v2.1-v2.5)
- v2.1: generated full-body runtime integration and deterministic path reliability baseline.
- v2.2: movement-balance safeguards improved.
- v2.3: session density/flow coherence improved.
- v2.4: role-based set assignment stabilized.
- v2.5: total generated set density escalated; underdosed plans improved, but muscle-volume distribution skew remained (arms/delts over-dominance in some profiles).

## Why a Derived Profile Layer Is Required
Directly consuming raw onboarding answers inside constructor logic creates drift and contradictory behavior. A deterministic `GenerationProfile` layer is required to:
- centralize dose/distribution calibration,
- enforce floors/caps consistently,
- keep weak-point personalization bounded,
- maintain explainability and reproducibility.

## Guardrails
- No runtime LLM logic.
- No gender-stereotype template branching.
- Anthropometry affects exercise fit, not core muscle-floor doctrine.
- Weak-point bias never overrides major floors or fatigue safety constraints.
