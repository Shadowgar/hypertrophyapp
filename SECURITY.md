# Security Policy

This document explains which versions of **Rocco's HyperTrophy** receive security updates and how security issues should be reported.

This project is currently maintained by a single developer. I appreciate responsible disclosure and will make every reasonable effort to investigate and address legitimate security issues.

---

## Supported Versions

Security fixes are provided for the most recent stable version of **Rocco's HyperTrophy**. Older versions are generally not maintained for security updates.

| Version | Supported |
| ------: | :-------- |
| `main` / current development | :white_check_mark: |
| Latest stable release | :white_check_mark: |
| Previous major releases | :x: |
| Older versions | :x: |

**Notes**

- I focus security updates on the latest version of the application.
- If a critical vulnerability is discovered, I may choose to patch older versions, but this is not guaranteed.
- Users are strongly encouraged to run the most recent version of the app.

---

## Reporting a Vulnerability

If you discover a security vulnerability in **Rocco's HyperTrophy**, please report it privately and **do not create a public GitHub issue**.

### Preferred Reporting Method

Use **GitHub Security Advisories**:

1. Go to the repository
2. Click the **Security** tab
3. Select **Report a vulnerability**

If this option is unavailable, you may contact me directly at:

**[ADD_CONTACT_EMAIL]**

---

### Information to Include

To help me investigate the issue efficiently, please include:

- A clear description of the vulnerability
- Steps required to reproduce the issue
- The version of the application affected
- Your testing environment (OS, runtime, browser, etc.)
- Any logs, screenshots, or proof-of-concept code if applicable

Please avoid exploiting the issue beyond what is necessary to demonstrate the problem.

---

### Response Expectations

Since this is an independently maintained project, response times may vary, but I will make a good-faith effort to follow this general timeline:

- **Acknowledgement:** within 72 hours
- **Initial review/update:** within 7 days
- **Fix timeline:** depends on severity and complexity

Critical vulnerabilities will be prioritized.

---

### Responsible Disclosure

I ask that security researchers allow time for the issue to be investigated and fixed before publicly disclosing the vulnerability.

If a report is confirmed:

- I will work on a fix
- A patch will be released
- A security advisory may be published with appropriate credit

If the issue cannot be reproduced or is determined not to be a security vulnerability, I will explain the reasoning.

---

## Safe Harbor

I support good-faith security research conducted responsibly. Please ensure that testing:

- Does not violate privacy
- Does not destroy or modify user data
- Does not degrade service availability
- Is limited to what is necessary to demonstrate the vulnerability

Thank you for helping keep **Rocco's HyperTrophy** secure.
